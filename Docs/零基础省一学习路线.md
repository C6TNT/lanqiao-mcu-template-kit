# 零基础省一学习路线

适用工程：
[蓝桥杯单片机通用模板工程](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程)

适用对象：

- 大一
- `C` 语言刚学
- 刚接触 `Keil`
- 目标是尽快达到“能独立做省赛题”的水平

这份路线不是按“先学很多原理再写代码”来排的，而是按“先把工程跑起来，再会改，再会限时做题”来排。

## 先看这个提醒

现在这套通用模板主工程本身已经做过比较多的编译和上板验证，稳定性是比较好的。

但是 `Examples` 里的历届省赛示例代码，更准确的定位是：

- 真题复盘版
- 模板迁移版
- 学习示例版

不要把它们默认理解成“每一届都已经完整上板验证、可以直接无检查使用的最终标准答案”。

尤其是：

- 第九届
- 第十届
- 第十一届

这几届更需要在学习和使用过程中边编译、边上板、边核对题面。

其实更稳妥的说法是：

- 目前所有 `Examples` 示例，都建议一边学习、一边验证代码正确性
- 如果发现和题面、现象、功能不一致，就继续反馈，再继续修改

这反而是最适合比赛训练的方式，因为比赛本身就要求你具备“边写边验证、边发现问题边修”的能力。

## 先说结论

如果目标是省一、进国赛，最重要的不是一开始把所有底层原理学得很深，而是尽快做到下面 4 件事：

1. 会打开工程、编译、烧录、看现象。
2. 会看懂模板里的 `App/app.c` 在干什么。
3. 会把 `Examples` 里的某一届题替换进主工程并跑通。
4. 会在 4 到 5 小时内独立完成一套历届省赛题。

只要这 4 步走通，后面刷题速度会越来越快。

## 学习总路线

建议分成 4 个阶段：

1. 工程入门阶段
2. 模板熟悉阶段
3. 真题套模板阶段
4. 限时冲刺阶段

## 第三阶段：真题套模板阶段

目标：

- 会从题面提取页面、参数、显示、按键、输出
- 会把一届省赛题套进当前模板
- 会对照示例自己重写

先看这里：

- [Examples/README.md](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Examples/README.md)

建议学习顺序：

1. 第十届
2. 第十二届
3. 第十三届
4. 第十一届
5. 第九届
6. 第十四届
7. 第十五届
8. 第十六届

每学一届，固定做这 4 件事：

1. 先看对应 `.md`，理解这题是怎么从基础模板长出来的。
2. 再看对应 `.c`，理解页面、按键、数据和输出是怎么挂到 `App_Loop()` 里的。
3. 自己不看代码，重新写一遍。
4. 编译验证，上板验证，再和示例对照补缺。

这一阶段一定要记住：

不要只“看懂”，要“跑通”。

## 第四阶段：限时冲刺阶段

目标：

- 从“会看会改”变成“规定时间内独立完成”
- 训练比赛真实节奏

这一阶段不要再只看代码，要按比赛流程来：

1. 先读题面。
2. 列出页面。
3. 列出参数。
4. 列出按键。
5. 列出输出灯和继电器。
6. 判断要用到哪些模块。
7. 再开始写 `app.c`。

推荐每次做题都按这个顺序：

1. 先写全局变量和页面枚举。
2. 再写显示函数。
3. 再写按键函数。
4. 再写数据采样和状态更新。
5. 最后挂到 `App_Init()` 和 `App_Loop()`。

不要一上来就先写最复杂的业务逻辑。先把页面切起来，题目就已经完成一半了。

## 最快拿省一的学习方法

对零基础同学来说，最有效率的方法不是“把所有模块都单独学很多遍”，而是“三遍法”：

第一遍：抄模板

- 目标是跑通，不求全懂。
- 重点是熟悉结构。

第二遍：看着题面自己改

- 目标是把示例题独立改出来。
- 重点是理解页面、按键、参数和输出之间的关系。

第三遍：脱离示例限时写

- 目标是接近比赛状态。
- 重点是时间管理和稳定性。

## 每周训练建议

如果离比赛还有一段时间，推荐一周节奏如下：

周一到周二：

- 看 1 届题的 `.md`
- 看 1 届题的 `.c`
- 自己重写 1 次

周三到周四：

- 再做 1 届题
- 对比示例修正

周五：

- 把这周两届题再脱稿写一遍

周末：

- 限时做 1 套完整题
- 复盘自己卡在哪里

## 比赛中最该避免的事

1. 一上来就纠结底层原理细节。
2. 数码管页面没写出来就先做复杂逻辑。
3. 按键还没通就开始怀疑所有模块都错了。
4. 一个功能没好就到处改底层。
5. 没有模板意识，每一题都想从零开始写。

## 比赛时最稳的做题顺序

推荐顺序：

1. 确认板子和工程正常。
2. 搭页面框架。
3. 实现按键切页。
4. 接入核心传感器数据。
5. 实现参数设置。
6. 实现 LED、继电器、DAC 等输出。
7. 补边界情况和异常状态。
8. 最后整体联调。

这套顺序的核心原因很简单：

- 页面和按键是骨架
- 传感器和参数是血肉
- 输出控制是收尾

骨架先立住，后面才不会乱。

## 对零基础最重要的一句话

不要把目标定成“我先把 51 单片机原理彻底学明白再比赛”，这样很容易拖慢节奏。

更现实、更高效的路线是：

- 先把这套模板用熟
- 再把历届题刷熟
- 在刷题过程中反过来补原理

对省赛来说，这样的效率通常更高。

## 最后的建议

如果时间有限，优先把下面这些练熟：

1. 页面切换
2. 参数设置
3. 数码管显示
4. 温度模块
5. `DS1302`
6. `PCF8591`
7. 频率测量
8. 超声波

如果这几块都能稳定接进当前模板，再把第十届到第十三届真正做顺，大概率就已经具备冲省一的基础了。

省一和省二很多时候差的不是“会不会某个模块”，而是：

- 工程够不够稳
- 结构够不够清楚
- 做题顺序对不对
- 刷题熟练度够不够

现在这套模板最大的价值，就是帮零基础同学把这 4 件事先稳下来。
