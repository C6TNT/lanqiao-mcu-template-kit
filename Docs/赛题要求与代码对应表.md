# 赛题要求与代码对应表

这份文档的目的很简单：

把“题面里的功能要求”直接对应到“模板里应该改哪里”。

这样看到题面时，不会只觉得内容很多，而是能马上定位到代码区域。

## 一、常见题面要求，对应改哪里

### 1. 要求切换多个页面

通常改：

- 页面枚举
- `App_ShowXXXPage`
- `App_UpdateDisplay`
- `App_HandleKey`

关键词一般是：

- 环境页
- 参数页
- 时间页
- 统计页
- 回显页

## 2. 要求设置参数

通常改：

- 参数结构体
- 参数默认值
- `App_FieldNext`
- `App_FieldAdd`
- `App_FieldSub`
- `App_SaveParam`
- `App_LoadParam`

关键词一般是：

- 温度参数
- 距离参数
- 超限参数
- 校准值

## 3. 要求读取温度

通常改：

- `g_data.temp`
- `DS18B20_ReadTemp()`
- 数据页 / 环境页显示函数
- 报警判断函数

关键词一般是：

- 温度检测
- 温度显示
- 高温报警

## 4. 要求读取时间

通常改：

- `g_data.rtc`
- `RTC_ReadTime()`
- `RTC_SetTime()`
- 时间页显示函数

关键词一般是：

- 时钟显示
- 整点触发
- 记录发生时间

## 5. 要求读取光敏、电位器、外部模拟量

通常改：

- `g_data.adc`
- `PCF8591_ReadADC(...)`
- 数据页或环境页显示函数

当前模板常见通道写法：

- 光敏电阻：`PCF8591_ReadADC(0x01)`
- 电位器：`PCF8591_ReadADC(0x03)`
- 外部 ADC 输入口：`PCF8591_ReadADC(0x05)`

## 6. 要求读取超声波距离

通常改：

- `g_data.dist`
- `Ultrasonic_ReadCm()`
- 运动页 / 记录页 / 状态页显示函数
- 接近判断逻辑

关键词一般是：

- 接近检测
- 距离检测
- 运动状态

## 7. 要求读取频率

通常改：

- `g_data.freq`
- `FREQ_PollTask()`
- `FREQ_GetHz()`
- 频率页显示函数
- 超限判断逻辑

关键词一般是：

- NE555
- 频率测量
- 校准值
- 最大频率

## 8. 要求 LED 指示

通常改：

- `App_UpdateAlarm`
- `App_Loop()` 里的输出部分
- `LED_Set()`

关键词一般是：

- 超限闪烁
- 状态指示
- 模式指示

## 9. 要求蜂鸣器 / 继电器动作

通常改：

- 报警判断
- `Beep_Set()`
- `Relay_Set()`
- `App_Loop()` 里的输出控制

关键词一般是：

- 报警
- 吸合
- 断开
- 提示音

## 10. 要求短按 / 长按 / 双击

通常改：

- `Key_GetEvent()`
- `Key_GetShortEvent()`
- `Key_GetDoubleEvent()`
- `Key_GetLongEvent()`
- `App_HandleKey()`

关键词一般是：

- 长按清零
- 双击切换
- 短按进入设置

## 二、按“做题顺序”看应该怎么改

### 第 1 步：先把页面名字和页面显示做出来

先改：

- 页面枚举
- `App_ShowXXXPage`
- `App_UpdateDisplay`

这样即使功能还没全实现，数码管也能先切页，能尽早拿到“页面分”。

### 第 2 步：再改按键逻辑

再改：

- `App_HandleKey`

先让：

- 能切页
- 能切子页
- 能进参数模式

### 第 3 步：再改参数逻辑

再改：

- 参数结构体
- 默认值
- 参数加减范围
- 参数保存与读取

### 第 4 步：再接采样功能

根据题目需要接：

- 温度
- RTC
- ADC
- 超声波
- 频率

### 第 5 步：最后补输出控制

最后改：

- LED
- 蜂鸣器
- 继电器
- DAC

因为这部分通常依赖前面的状态判断已经成立。

## 三、看题面时的一个实用方法

读题时不要直接想“这题我不会写”。

更稳的办法是把题目拆成 5 类：

1. 页面显示要求
2. 按键要求
3. 参数要求
4. 采样要求
5. 输出控制要求

拆完以后，再回到模板里一一对应去改，思路会清楚很多。

## 四、最关键的一句话

比赛里很多综合题看起来复杂，本质上都可以拆成：

- 页面怎么显示
- 按键怎么切换
- 参数怎么保存
- 数据怎么采样
- 条件满足后输出怎么动作

只要能把题面先拆成这几类，再回到模板里找对应区域，读代码和改代码都会轻松很多。
