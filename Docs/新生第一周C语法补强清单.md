# 新生第一周 C 语法补强清单

这份清单不是大学 C 语言完整课程，而是专门为了让你在最短时间内：

- 能看懂模板
- 能改 `App/app.c`
- 能开始刷蓝桥杯单片机真题

目标不是“把 C 学得很系统”，而是先进入“能改题”的状态。

---

## 第 1 天：先把变量和函数看懂

今天只做这两件事：

1. 看 [模板里最常见的C语法.md](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Docs/模板里最常见的C语法.md)
   重点看：
   - 变量
   - 函数
   - `u8 / u16 / bit`
   - `static`

2. 打开 [app.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/App/app.c)
   只做一件事：
   - 把所有全局变量看一遍
   - 大概知道每个变量在存什么

今天结束前你至少应该知道：
- `g_page` 是当前页面
- `g_param` 是参数
- `g_data` 是实时数据
- `g_alarm` 是报警状态

---

## 第 2 天：先把 if / switch 看懂

今天重点补：
- `if / else`
- `switch / case`

建议这样看：

1. 继续看 [模板里最常见的C语法.md](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Docs/模板里最常见的C语法.md)
2. 再看 [从C语法到赛题代码的映射.md](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Docs/从C语法到赛题代码的映射.md)

然后打开 [app.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/App/app.c)，重点看：
- `App_ShowParamPage()`
- `App_HandleUart()`
- 报警判断相关逻辑

今天结束前你至少应该知道：
- `if` 最常拿来做判断
- `switch` 最常拿来做页面或参数切换

## 第 3 天：先把结构体和枚举看懂

今天重点补：
- `typedef struct`
- `typedef enum`
- 结构体成员访问 `.` 

建议看：
- [新生最容易看不懂的代码写法.md](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Docs/新生最容易看不懂的代码写法.md)

然后重点对照 [app.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/App/app.c) 里的这些定义：
- `app_page_t`
- `app_field_t`
- `app_data_t`
- `app_param_t`

今天结束前你至少应该知道：
- 为什么页面要用 `enum`
- 为什么参数和数据要用结构体
- `g_param.temp_limit_x10` 这种写法到底是什么意思

---

## 第 4 天：先把显示代码看懂

今天重点不是新语法，而是练“看懂一段代码在干嘛”。

重点看：
- `App_ShowDataPage()`
- `App_ShowParamPage()`
- `App_ShowTimePage()`
- `App_ShowFreqPage()`
- `App_ShowRecordPage()`

你今天只做一件事：
- 把每个显示函数写一句中文解释

例如：
- `App_ShowDataPage()`：显示温度和 ADC
- `App_ShowFreqPage()`：显示频率

今天结束前你至少应该知道：
- 数码管显示代码不是乱写的
- 每个页面函数只负责一种页面

---

## 第 5 天：先把按键逻辑看懂

今天重点看：
- `App_HandleKey()`
- [bsp_key.h](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/BSP/bsp_key.h)

要搞清楚：
- 普通按键事件怎么拿
- 短按 / 双击 / 长按怎么拿
- 为什么按键功能要放在应用层，不写在底层

今天结束前你至少应该知道：
- 底层只负责“识别哪个键被按了”
- 业务层才负责“这个键要干什么”

---

## 第 6 天：先把 App_Loop 看懂

今天重点看：
- `App_Loop()`
- 各种节拍标志

重点理解：
- `10ms` 做什么
- `100ms` 做什么
- `500ms` 做什么
- `1s` 做什么

今天结束前你至少应该知道：
- 单片机程序不是“从上到下只跑一次”
- 而是主循环一直跑
- 不同任务按不同时间节拍执行

---

## 第 7 天：第一次自己改一个小功能

今天不要再只看文档，要开始动手。

推荐练习：

1. 改默认参数  
   比如把：
   - 温度参数从 `30.0°C` 改成 `28.0°C`
   - 距离参数从 `30cm` 改成 `40cm`

2. 改一个页面显示
   比如让某页多显示一个字符

3. 改一个按键行为
   比如按键切换顺序改一下

练习顺序建议：
1. 先改变量
2. 再编译
3. 再烧录
4. 再看结果对不对

---

## 第一周最重要的目标

这一周结束后，你不需要达到“自己从零写完整题”的程度。

你真正应该达到的是：

- 看懂模板的大体结构
- 知道页面、按键、参数、采样、报警分别在哪写
- 能改简单功能
- 能开始对着示例题刷题

这就已经够了。

---

## 第一周最容易犯的错

1. 一上来就想把所有底层原理学懂  
结果时间花很多，但还不会改题。

2. 一上来就想背整套代码  
结果背得很累，但不知道为什么这么写。

3. 不编译、不上板，只看代码  
这样进步会很慢。

4. 一看到结构体、枚举、`static` 就害怕  
这些其实都是模板里很固定的写法。
