# 蓝桥杯单片机通用模板工程使用说明

这套模板现在已经按综合题常见的资源冲突做过一次重新分配，目标是让：

- 串口可以开
- 独立按键可以用
- 温度、ADC、RTC、超声波可以一起跑
- 频率读取也能加入默认工程

## 一、当前模板的资源分配

当前默认工程的资源分配是：

- 主系统时基：`Timer1`
- 频率计数：`Timer0`
- 串口波特率：`Timer2`
- 超声波计时：`PCA`

这样做的原因是：

- `Timer1` 不再被串口占用，主循环节拍更稳定
- `Timer0` 可以专门留给频率题
- `Timer2` 用来做串口，不再和超声波抢资源
- 超声波改成 `PCA` 计时后，可以和串口、频率共存

这套思路参考了这篇文章里的分配方式：
[蓝桥杯单片机模板](https://blog.csdn.net/2301_80545197/article/details/136601822)

当前默认 [app.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/App/app.c) 已经实现了：

## 三、当前页面说明

默认一共有 5 个页面：

### 1. 数据页 `PAGE_DATA`

显示内容：

- 左侧温度
- 右侧 ADC

说明：

- 上电前约 800ms 温度未准备好时显示 `C----`
- 温度准备好后显示真实温度
- 右侧持续显示 ADC 数值

### 2. 参数页 `PAGE_PARAM`

可以查看和修改 3 个参数：

- 温度报警阈值 `temp_limit_x10`
- 距离报警阈值 `dist_limit`
- ADC 报警阈值 `adc_limit`

说明：

- `temp_limit_x10 = 300` 表示 `30.0°C`
- 参数页最右侧显示 `A` 时，表示当前处于编辑状态

### 3. 时间页 `PAGE_TIME`

显示格式：

- `hh-mm-ss`

默认上电时间：

- `12:00:00`

### 4. 频率页 `PAGE_FREQ`

显示格式：

- 第 1 位显示 `F`
- 后 7 位显示当前频率值，单位 `Hz`

说明：

- 当前频率使用 `Timer0` 外部计数
- 默认从 `T0(P3.4)` 输入脉冲
- 采用 `250ms` 门控换算
- 应用层做了一次简单平滑，所以显示更稳

### 5. 记录页 `PAGE_RECORD`

显示内容：

- 超声波距离
- 报警标记
- 参数模式标记

说明：

- 超声波超时显示 `999`
- 距离显示的是平滑后的结果，不是单次裸值

## 四、按键说明

当前默认业务层使用独立按键 `1~4`：

### 普通模式

- `key1`：循环切换页面
- `key2`：在参数页进入编辑模式
- `key3`：默认无操作
- `key4`：默认无操作

### 参数编辑模式

- `key1`：切换参数项
- `key2`：保存并退出
- `key3`：参数加
- `key4`：参数减

## 五、报警怎么触发

只要满足下面任意一条，就进入报警状态：

- 当前温度大于温度阈值
- 当前距离小于距离阈值
- 当前 ADC 大于 ADC 阈值

报警输出方式：

- `LED0` 闪烁
- 蜂鸣器闪烁

报警闪烁节拍：

- `500ms`

## 六、各时间任务负责什么

### 10ms 任务

- 扫描按键
- 生成按键事件
- 处理页面切换和参数编辑

### 100ms 任务

- 读取 ADC
- 读取温度
- 读取超声波
- 更新距离平滑值
- 更新报警状态

### 500ms 任务

- 读取一次频率
- 更新频率平滑值
- 驱动 LED 和蜂鸣器报警闪烁

### 1s 任务

- 读取 RTC 时间
- 串口发送调试信息

## 七、串口说明

当前模板里：

- 串口仍然走 `P3.0/P3.1`
- 串口波特率发生器改成了 `Timer2`

这样做的好处是：

- 接线和常见开发板默认串口一致
- 不再和超声波抢 `Timer1`
- 默认不做高频周期输出
- 收到一个字符后才返回一组数据

### 当前默认串口交互方式

上电后串口会发送一行提示：

- `UART ready. Send '?' for snapshot.`

之后可以这样测试：

- 发送 `?`
  返回一组当前数据

- 发送 `s`
  也返回一组当前数据

- 发送 `p`
  返回当前参数

- 发送 `h`
  返回帮助提示

### 当前默认返回内容

发送 `?` 或 `s` 后，会回类似下面这一行：

- `T=26.5,ADC=132,DIST=48,FREQ=1000`

含义分别是：

- `T`：当前温度
- `ADC`：当前 ADC 结果
- `DIST`：当前超声波距离
- `FREQ`：当前频率值

发送 `p` 后，会回类似下面这一行：

- `TL=30.0,DL=30,AL=200`

含义分别是：

- `TL`：温度阈值
- `DL`：距离阈值
- `AL`：ADC 阈值

## 八、超声波说明

当前超声波模块不再使用 `Timer1`，而是改成了 `PCA` 计时。

这样可以和下面这些模块同时工作：

- 串口
- 系统节拍
- 频率读取

如果后面题目是纯超声波题，也可以单独把超声波逻辑从 [dev_ultrasonic.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Devices/dev_ultrasonic.c) 拿出去精简。

## 九、频率说明

当前频率模块：

- 使用 `Timer0` 外部计数模式
- 默认输入口是 `T0(P3.4)`
- 每次统计 `250ms`
- 最终乘 4 换算成 `Hz`

为了让显示更稳，默认业务层又做了一次平滑处理，所以页面看到的是平滑后的频率值。

## 十、ADC 通道参数怎么填

当前 [PCF8591_ReadADC](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Devices/dev_pcf8591.h) 的参数 `channel` 已经按常见板卡接法补了说明。

最常用的 3 个写法是：

- 采样光敏电阻：`PCF8591_ReadADC(0x01)`
- 采样滑动电位器：`PCF8591_ReadADC(0x03)`
- 采样外部真实 ADC 输入口：`PCF8591_ReadADC(0x05)`

如果后面题目写的是：

- “光敏电阻电压”  
  就优先用 `0x01`

- “电位器输出电压”  
  就优先用 `0x03`

- “ADC 口输入电压” 或“外部模拟量输入”  
  就优先用 `0x05`

## 十三、高级按键事件说明

当前模板除了原来的普通按下事件，还补充了：

- 单击事件
- 双击事件
- 长按事件

对应接口在：

- [bsp_key.h](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/BSP/bsp_key.h)
- [bsp_key.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/BSP/bsp_key.c)

现在可用的按键接口有：

- `Key_GetEvent()`
- `Key_GetShortEvent()`
- `Key_GetDoubleEvent()`
- `Key_GetLongEvent()`

含义分别是：

- `Key_GetEvent()`：按下立即产生事件，兼容旧模板逻辑
- `Key_GetShortEvent()`：确认不是双击后的单击事件
- `Key_GetDoubleEvent()`：双击事件
- `Key_GetLongEvent()`：长按事件

默认阈值是：

- 长按阈值：`25 x 10ms = 250ms`
- 双击窗口：`40 x 10ms = 400ms`

默认主工程里已经做了一个最小演示：

- 记录页会显示最近一次高级按键事件
- `1` 表示单击
- `2` 表示双击
- `3` 表示长按
- 后两位显示对应按键编号

也就是说，在默认工程里切到记录页后，直接测试按键，就能看到双击和长按是否被识别到。

如果后面某套真题需要：

- 长按清零
- 双击切换模式
- 单击和双击区分不同功能

就建议优先使用这组高级接口，而不是只使用原来的 `Key_GetEvent()`。
