# 新手最短阅读入口

如果你是第一次打开这套工程，不要试图一次看完所有内容。

最短、最稳的阅读路径就是下面这 5 步。

## 第 1 步：先看使用说明

[使用说明.md](./使用说明.md)

这一步的目标是先知道：

- 当前模板实现了什么
- 页面、按键、报警、串口分别在做什么
- 这套工程默认能跑哪些模块

## 第 2 步：再看源码导读

[模板源码导读.md](./模板源码导读.md)

这一步的目标是先知道：

- `main.c` 干什么
- `App/app.c` 干什么
- `BSP / Devices / Drivers` 各自干什么
- 代码应该从哪里开始读

## 第 3 步：再看赛题要求与代码对应表

[赛题要求与代码对应表.md](./赛题要求与代码对应表.md)

这一步的目标是先知道：

- 题目要求切页面时改哪里
- 题目要求调参数时改哪里
- 题目要求接温度、频率、超声波时改哪里
- 题目要求做 LED、继电器、DAC 输出时改哪里

## 如果你 C 语法基础比较弱，先插看这 3 份

[模板里最常见的C语法.md](./模板里最常见的C语法.md)

[新生最容易看不懂的代码写法.md](./新生最容易看不懂的代码写法.md)

[从C语法到赛题代码的映射.md](./从C语法到赛题代码的映射.md)

这 3 份的作用分别是：
- 第一份：帮你把最常见的语法先认出来
- 第二份：帮你看懂模板里最容易把人看懵的写法
- 第三份：帮你把语法和赛题要求对上

如果你希望第一周不只是“看”，还想知道每天到底练什么，再接着看这份：

如果你想边学边练，再配合这份：

## 第 4 步：再看主业务层代码

[App/app.c](../App/app.c)

这一步不要逐行硬背，先按下面顺序看：

1. 全局变量
2. `App_ShowXXXPage`
3. `App_HandleKey`
4. `App_UpdateAlarm`
5. `App_Loop`

## 第 5 步：最后再进 Examples

[Examples/README.md](../Examples/README.md)

看某一届示例时，推荐顺序是：

1. 先看该届 `.md`
2. 再看该届 `.c`
3. 最后自己回到 `App/app.c` 改一遍

## 最后提醒

如果你现在时间很紧，只想先能刷题，不要一开始就把所有底层原理搞懂。

更稳的路线是：

1. 先会编译和烧录
2. 先会看页面和按键逻辑
3. 先会改参数和显示
4. 再慢慢补外设原理

这套模板的目标，本来就是先帮你尽快进入“能改题”的状态。
