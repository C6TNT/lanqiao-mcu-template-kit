# 新生最容易看不懂的代码写法

这份文档专门解释：第一次看这套模板时，哪些写法最容易把人看懵。

不是你笨，而是这些写法本来就不是普通教材里最先讲的。

## 1. `static app_param_t g_param = {300, 30, 200};`

这句可以拆成 4 部分理解：

- `static`
  说明这个变量只在当前 `app.c` 文件里用

- `app_param_t`
  这是变量类型，表示“参数结构体”

- `g_param`
  这是变量名

- `{300, 30, 200}`
  这是初始值

对应关系是：

- `300` -> 温度参数 `30.0°C`
- `30` -> 距离参数 `30cm`
- `200` -> ADC 参数 `200`

## 2. `typedef struct`

这个组合非常常见。

例如：

```c
typedef struct
{
    float temp;
    u8 adc;
    u16 dist;
    u16 freq;
    rtc_time_t rtc;
} app_data_t;
```

你可以理解成：
- 先定义一包数据长什么样
- 再给这包数据起个名字叫 `app_data_t`

以后就可以直接写：

```c
app_data_t g_data;
```

## 3. `typedef enum`

这个常用来表示“有限个状态”。

例如：

```c
typedef enum
{
    PAGE_DATA = 0,
    PAGE_PARAM,
    PAGE_TIME
} app_page_t;
```

你可以把它理解成：
- 页面有多个状态
- 但不想只用 `0/1/2` 这种看不懂的数字
- 所以给它们起了名字

## 4. `g_` 前缀

模板里很多变量都长这样：

- `g_page`
- `g_param`
- `g_data`
- `g_alarm`

这里的 `g_` 一般表示：
- global，全局变量

你看到 `g_`，就要想到：
- 这个变量通常不是函数里临时用一下
- 它是整个 `app.c` 都会反复用的状态

## 5. `App_` 前缀

模板里很多函数都长这样：

- `App_Init`
- `App_Loop`
- `App_ShowDataPage`
- `App_HandleUart`

这里的 `App_` 表示：
- 这些函数属于应用层
- 它们负责“赛题逻辑”

简单说：
- `BSP/Drivers/Devices` 更偏底层
- `App` 更偏题目功能实现

## 6. `u8`、`u16`

很多新生第一次看到会怕，其实不用怕。

你就先这样记：

- `u8`：小整数
- `u16`：大一点的整数

在模板里通常：
- `u8` 用来存按键值、ADC 值、小参数
- `u16` 用来存距离、频率、计数

## 7. `bit`

例如：

```c
static bit g_alarm = 0;
```

意思就是：
- `g_alarm` 只有两种状态
- `0` 表示没报警
- `1` 表示报警

你可以把它看成“开关量”。

## 8. `void`

常见写法：

```c
void App_Init(void)
```

这里两个 `void` 的意思不一样：

- 前面的 `void`
  表示这个函数“没有返回值”

- 后面的 `void`
  表示这个函数“没有参数”

## 9. `switch(g_field)`

很多新生看到 `switch` 会绕。

最简单的理解：
- `if` 是判断 1 种或 2 种情况
- `switch` 是判断很多种情况

比如参数页：

```c
switch(g_field)
{
case FIELD_TEMP_LIMIT:
    ...
    break;

case FIELD_DIST_LIMIT:
    ...
    break;
}
```

意思是：
- 如果当前选中温度参数，就按温度参数显示
- 如果当前选中距离参数，就按距离参数显示

## 10. `g_param.temp_limit_x10`

这个点特别容易把新生看懵。

你这样理解就行：

- `g_param` 是“一整包参数”
- `temp_limit_x10` 是这包参数里的“温度参数”

中间那个点 `.` 的意思就是：
- 取结构体里的某个成员

所以：

```c
g_param.temp_limit_x10
```

意思就是：
- 取参数结构体里的温度阈值

## 11. `temp_limit_x10`

为什么后面有个 `_x10`？

意思是：
- 这个温度不是直接按浮点数存
- 而是放大 10 倍再存

例如：
- `300` 表示 `30.0°C`
- `235` 表示 `23.5°C`

这样做是为了：
- 方便单片机处理
- 少用浮点

## 12. `SEG_SetCode()` 和 `SEG_SetDigit()`

很多新生刚看到数码管函数也会发懵。

你可以这样分：

- `SEG_SetCode`
  直接放“字符/符号”

- `SEG_SetDigit`
  直接放数字

比如：

```c
SEG_SetCode(0, SEG_C);
SEG_SetDigit(1, 3);
```

意思是：
- 第 0 位显示字母 `C`
- 第 1 位显示数字 `3`

## 13. `App_Loop()` 里一堆节拍判断

新生很容易看不懂这种：

```c
if(g_flag_10ms)
{
    ...
}

if(g_flag_100ms)
{
    ...
}
```

你可以理解成：
- `10ms` 做按键扫描、轻任务
- `100ms` 做采样、刷新
- `500ms` 做闪烁、报警节拍
- `1s` 做慢速任务

也就是：
- 不是所有事情都每圈都做
- 而是按时间分开做

## 14. 看不懂时该怎么办

推荐顺序：

1. 先看变量定义
2. 再看页面显示函数
3. 再看按键函数
4. 最后看 `App_Loop`

不要一上来从头到尾硬啃。
