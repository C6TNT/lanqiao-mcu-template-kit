# Keil常见报错速查表

这份文档是给刚开始接触 `Keil` 的同学准备的。

原则只有一个：

**先看第一条报错，不要一上来就被后面一串连带报错吓住。**

## 1. `syntax error near ...`

### 报错是什么意思

代码在某个位置附近语法不合法。

### 最常见原因

- 少了分号 `;`
- 少了右括号 `)` 或右花括号 `}`
- 函数参数列表写错
- 注释或字符串没有正常结束

### 先检查哪里

1. 报错行的上一行
2. 报错行所在函数开头
3. 最近刚改动过的括号和分号

## 2. `too many actual parameters`

### 报错是什么意思

调用函数时传进去的参数个数，比函数声明的参数个数多。

### 最常见原因

- `.c` 里改了函数定义，但 `.h` 里的声明没同步改
- 看错了函数原型
- 复制别的题代码时参数没对上

### 先检查哪里

1. 当前函数调用位置
2. 对应 `.h` 文件里的函数声明
3. 对应 `.c` 文件里的函数定义

## 3. `undefined identifier`

### 报错是什么意思

编译器找不到这个变量名、函数名、宏名或类型名。

### 最常见原因

- 名字拼错
- 头文件没包含
- 变量没有定义
- 宏没有写到 `.h` 里

### 先检查哪里

1. 名字有没有拼错
2. 对应头文件有没有 `#include`
3. 这个变量或函数是不是根本还没写

## 4. `multiple definition`

### 报错是什么意思

同一个变量或函数被定义了不止一次。

### 最常见原因

- 在 `.h` 里直接定义了变量
- 同一个函数复制了两份
- 一个文件被重复加入工程

### 先检查哪里

1. 是不是把变量定义写进了头文件
2. `Keil` 工程里有没有重复加文件
3. 搜索同名变量或函数出现了几次

## 5. `cannot open source file`

### 报错是什么意思

找不到你要包含的头文件。

### 最常见原因

- 文件路径不对
- 头文件名拼错
- 文件根本不在工程目录里

### 先检查哪里

1. `#include "xxx.h"` 这个名字有没有写对
2. 文件是不是确实存在
3. 工程包含路径有没有配好

## 6. `Target not created`

### 报错是什么意思

目标文件没有生成。

### 最常见原因

- 前面已经有编译错误
- 某个目标文件没有成功生成

### 先检查哪里

1. 往上翻，找到第一条真正的 `error`
2. 不要只盯着最后这句

## 7. `object file not found`

### 报错是什么意思

链接阶段找不到某个 `.obj` 文件。

### 最常见原因

- 前面那个 `.c` 文件编译失败了
- 文件没被正确加入工程

### 先检查哪里

1. 对应 `.c` 文件前面有没有编译错误
2. 文件是不是在 `Keil` 工程里

## 8. `cannot open file ...`

### 报错是什么意思

链接器或编译器找不到某个文件。

### 最常见原因

- 文件路径错误
- 文件名改了但工程没同步
- 工程引用了已经删除的文件

### 先检查哪里

1. 工程树里对应文件还在不在
2. 文件名是不是改过
3. 是否引用了已删除文件

## 9. `redefinition`

### 报错是什么意思

变量、函数或类型被重复定义。

### 最常见原因

- 同一个名字写了两遍
- 复制代码时忘记改名
- 头文件保护没写好

### 先检查哪里

1. 搜索同名内容
2. 看是不是复制了旧代码没删干净

## 10. `illegal indirection`

### 报错是什么意思

对不该取地址或不该解引用的对象做了 `*` 或 `&` 操作。

### 最常见原因

- 把普通变量当指针用
- 变量类型和函数参数类型不匹配

### 先检查哪里

1. 函数参数类型
2. 当前变量定义类型

## 11. `bit member in struct/union`

### 报错是什么意思

`Keil C51` 不允许把 `bit` 放进结构体成员里。

### 最常见原因

- 参考了普通 C 写法
- 把状态位直接塞进结构体

### 先检查哪里

1. 当前结构体定义
2. 是否把 `bit` 改成 `u8`

## 12. 一串报错同时出现怎么办

优先原则：

1. 只看第一条 `error`
2. 修完后重新编译
3. 再看新的第一条 `error`

不要一次想把后面十几条一起修完，因为很多都是前面一条引起的连带报错。
