# 社团群公告版

各位准备蓝桥杯单片机的同学看这里。

我这边整理了一套基于当前稳定工程的训练资料，适合零基础和刚入门的同学用来备赛。

先说明一下定位：

- 主模板工程本身已经做过较多编译和上板验证，稳定性比较好。
- `Examples` 里的历届示例代码主要是学习版、复盘版、模板迁移版。
- 不建议直接把 `Examples` 当成最终标准答案无脑套用。

更稳妥的做法是：

- 边看
- 边改
- 边编译
- 边上板验证
- 发现问题继续修

这本身也是比赛最需要的能力。

## 建议学习顺序

先把这些最基础的动作做熟：

1. 打开工程
2. 编译
3. 烧录
4. 跑通默认模板

然后再去看主工程里的：

- [app.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/App/app.c)

先弄清楚：

- 页面在哪写
- 按键在哪写
- 参数在哪写
- 输出控制在哪写

后面再按顺序学习 `Examples`，推荐顺序是：

1. 第十届
2. 第十二届
3. 第十三届
4. 第十一届
5. 第九届
6. 第十四届
7. 第十五届
8. 第十六届

每学一届，固定按这个流程来：

1. 先读对应 `.md`
2. 再看对应 `.c`
3. 自己不看答案重写一遍
4. 编译验证
5. 上板验证

## 如果时间比较紧

训练重点优先放在这些：

1. 页面切换
2. 参数设置
3. 数码管显示
4. `DS18B20`
5. `DS1302`
6. `PCF8591`
7. 频率测量
8. 超声波

## 比赛里最稳的做题顺序

尽量固定按这个顺序写：

1. 先搭页面框架
2. 再做按键切页
3. 再接数据采样
4. 再做参数设置
5. 最后补 `LED`、继电器、`DAC`、串口这些输出控制

不要一上来就写最复杂的业务逻辑。

也不要一个地方不对就推翻整个工程重写。

## 最后提醒

第九届、第十届、第十一届这些示例尤其建议边学习边验证。

其实现在所有示例代码都建议在使用过程中：

- 边编译
- 边上板
- 边核对题面

如果在使用过程中发现：

- 页面不一致
- 按键不一致
- 显示不一致
- 输出逻辑不一致

直接把问题和现象发我，我再进行维护修正。
