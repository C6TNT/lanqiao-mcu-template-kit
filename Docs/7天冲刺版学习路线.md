# 7天冲刺版学习路线

适用对象：

- 只剩 7 天左右备赛时间
- 需要尽快把模板跑熟
- 目标是尽量稳定拿分，冲省一

适用工程：
[蓝桥杯单片机通用模板工程](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程)

## 先看这个提醒

这 7 天冲刺版的核心目标不是“把所有原理学透”，而是“尽快形成比赛作战能力”。

另外要先说明：

- 主模板工程已经做过较多验证，可以作为主学习工程
- `Examples` 里的历届示例代码主要是学习版、复盘版、模板迁移版
- 不是所有示例都已经做过完整上板逐项验证

所以这 7 天里要采用的策略是：

- 一边看示例
- 一边替换进主工程
- 一边编译
- 一边上板验证
- 发现问题就记录并继续修

不要把 `Examples` 直接当“绝对正确答案”背下来不验证。

## 7 天总原则

1. 每天都要上板。
2. 每天都要编译。
3. 每天都要手写 `app.c` 的一部分。
4. 每天都要做至少一次限时练习。

## 第 1 天：把工程跑通

目标：

- 会打开工程
- 会编译
- 会烧录
- 会看默认模板现象

今天只做这些：

1. 看 [Keil与烧录入门.md](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Docs/Keil与烧录入门.md)
2. 看 [使用说明.md](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Docs/使用说明.md)
3. 打开主工程
4. 编译
5. 烧录
6. 把默认模板的页面和按键操作摸清楚

今天结束前必须做到：

- 不需要别人帮忙就能重新编译和烧录一次

## 第 6 天：完整限时做一套

目标：

- 开始进入比赛状态

今天建议：

1. 选一套相对熟悉的题
2. 严格限时
3. 不看答案先做
4. 做完后再对照 `Examples`

今天最重要的不是“做对全部”，而是找到自己最容易卡住的地方。

常见卡点一般是：

- 页面没先搭起来
- 按键逻辑混乱
- 参数和输出耦合太乱
- 想一步到位，结果越写越乱

## 第 7 天：查漏补缺 + 模拟比赛

目标：

- 把最常见错误提前踩完

今天建议做两件事：

1. 再完整限时做一套
2. 把自己前 6 天出现过的错误再复盘一遍

今天最该检查的是：

- 工程能不能稳定编译
- 烧录流程会不会出错
- 页面切换是否稳定
- 按键是否稳定
- 显示是否稳定
- 传感器数据是否正常
- 参数设置是否正常
- 输出灯、继电器、DAC 是否符合题意

## 这 7 天最该背下来的东西

1. `App_Init()` 的写法
2. `App_Loop()` 的调度结构
3. 页面枚举怎么写
4. 按键处理怎么写
5. 显示函数怎么写
6. 参数页面怎么写
7. 输出控制怎么写

## 7 天里最不该做的事

1. 天天只看视频不动手
2. 天天只抄代码不上板
3. 看到错就怀疑整个模板都不行
4. 一题还没写完就换下一题
5. 总想把所有底层原理先学完再开始

## 最后建议

如果只剩 7 天，最优先的练习顺序还是：

1. 第十届
2. 第十二届
3. 第十三届
4. 第十四届或第十五届

原因很简单：

- 第十届练频率主线
- 第十二届练温度和参数
- 第十三届练 RTC 和模式切换
- 第十四、十五届练复杂页面和综合业务

这 7 天冲刺的核心不是“每届都完全吃透”，而是：

- 把主模板用熟
- 把最常见题型做顺
- 把比赛流程练顺

做到这三点，零基础同学也有机会在比赛里把分数稳定做上去。
