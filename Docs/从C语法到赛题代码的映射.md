# 从C语法到赛题代码的映射

很多新生学了 C 语法以后，还是不会做题。  
原因不是语法没学，而是不会把语法映射到赛题。

这份文档就是专门解决这个问题的。

## 1. 题目要求“切换页面”

你真正会用到的语法：
- `enum`
- 变量
- `if / switch`

模板里对应的代码位置：
- 页面枚举：`app_page_t`
- 当前页面变量：`g_page`
- 页面显示函数：`App_ShowDataPage()`、`App_ShowParamPage()` 等

你可以这样理解：
- `enum` 负责给页面编号
- `g_page` 负责记录当前在哪个页面
- 按键函数里修改 `g_page`
- 显示函数里根据 `g_page` 决定显示什么

## 2. 题目要求“设置参数”

你真正会用到的语法：
- 结构体
- 变量
- `switch`
- 加减运算

模板里对应的代码位置：
- 参数结构体：`app_param_t`
- 当前参数变量：`g_param`
- 当前选中参数：`g_field`
- 参数存储：`App_SaveParam()`、`App_LoadParam()`

你可以这样理解：
- 结构体负责“把一组参数打包”
- `g_field` 负责“现在在改哪一个参数”
- 按键决定参数加减
- EEPROM 函数决定掉电保存

## 3. 题目要求“报警判断”

你真正会用到的语法：
- `if / else`
- 比较运算
- 状态变量

模板里对应的代码位置：
- `g_alarm`
- `g_alarm_flash`
- `App_UpdateAlarm()`

你可以这样理解：
- 先写判断条件
- 再把结果存进报警标志
- 再用 LED / 蜂鸣器 / 继电器去表现这个状态

## 4. 题目要求“显示温度、ADC、频率、距离”

你真正会用到的语法：
- 变量
- 数字拆位
- 函数调用

模板里对应的代码位置：
- `g_data`
- `App_ShowDataPage()`
- `App_ShowFreqPage()`
- `App_ShowRecordPage()`

你可以这样理解：
- 传感器采样得到数据，先存到 `g_data`
- 显示页再从 `g_data` 里取出来
- 数码管函数只负责“显示”

## 5. 题目要求“按键实现功能”

你真正会用到的语法：
- `if`
- `switch`
- 状态变量

模板里对应的代码位置：
- `App_HandleKey()`
- `Key_GetEvent()`
- `Key_GetShortEvent()`
- `Key_GetDoubleEvent()`
- `Key_GetLongEvent()`

你可以这样理解：
- 底层先告诉你“哪个键被按了”
- 应用层再决定“这个键要做什么”

## 6. 题目要求“多个界面循环切换”

你真正会用到的语法：
- 枚举
- 加减
- 越界回绕

典型思路：

```c
g_page++;
if(g_page > PAGE_RECORD)
{
    g_page = PAGE_DATA;
}
```

意思是：
- 页面号加 1
- 超过最后一页后再回到第一页

## 7. 题目要求“动态显示数码管”

你真正会用到的语法：
- 数组
- 函数

模板里对应的模块：
- `BSP/bsp_seg.c`
- `SEG_SetCode()`
- `SEG_SetDigit()`
- `SEG_ClearBuffer()`

你可以这样理解：
- 业务层负责告诉数码管“每一位要显示什么”
- 底层负责把它动态扫描出来

所以刷题时你一般改的是“显示内容”，不是底层扫描逻辑。

## 8. 题目要求“定时做任务”

你真正会用到的语法：
- `if`
- 状态标志

模板里对应的代码位置：
- `g_flag_10ms`
- `g_flag_100ms`
- `g_flag_500ms`
- `g_flag_1s`
- `App_Loop()`

你可以这样理解：
- `10ms` 任务：按键、轻量轮询
- `100ms` 任务：采样、页面刷新
- `500ms` 任务：闪烁、节拍
- `1s` 任务：慢速统计、时间更新

## 9. 题目要求“掉电保存参数”

你真正会用到的语法：
- 函数
- 变量
- 结构体成员访问

模板里对应的代码位置：
- `App_SaveParam()`
- `App_LoadParam()`
- `AT24C02_WriteByte()`
- `AT24C02_ReadByte()`

你可以这样理解：
- 结构体里放的是“内存中的参数”
- EEPROM 里放的是“掉电后还能记住的参数”

## 10. 题目要求“多个状态切换”

比如：
- 浏览模式 / 编辑模式
- 正常状态 / 报警状态
- 温控模式 / 时控模式

你真正会用到的语法：
- 状态变量
- `if`
- 枚举

模板里对应的例子：
- `g_param_mode`
- `g_alarm`

你可以这样理解：
- 先定义“当前处于什么模式”
- 再根据模式决定按键行为和显示内容

## 11. 题目要求“显示字符和数字混合”

你真正会用到的语法：
- 函数调用
- 数字拆位

模板里典型写法：

```c
SEG_SetCode(0, SEG_C);
SEG_SetDigit(1, 3);
SEG_SetDigit(2, 0);
```

意思是：
- 第 0 位显示字符
- 后面几位显示数字

这类题在蓝桥杯里非常常见。

## 12. 最重要的一条：赛题不是在考你“语法题”

大多数赛题的本质不是：
- “你会不会写 `if`”
- “你会不会定义结构体”

而是：
- 你能不能把页面、按键、参数、显示、采样、输出拆成几块
- 再用合适的 C 语法把它们拼起来

所以你学模板时，要始终问自己：

- 这段代码在做页面、按键、参数，还是输出？
- 它对应赛题里的哪一条要求？
- 我如果要改题，应该改哪一块？

只要这个映射慢慢建立起来，你做题速度会快很多。
