# 报错反馈模板

如果你在使用这套工程时编译不过，建议按下面格式反馈。

这样别人帮你排错会快很多。

## 推荐反馈格式

```text
1. 报错文件：

2. 第一条 error 原文：

3. 我刚刚改了哪个文件：

4. 我改的是哪一部分：
   例如：页面、按键、参数、频率、超声波、显示

5. 现在主工程还是示例工程：
   例如：我把 app_provincial_10.c 替进了 App/app.c

6. 我已经做过的检查：
   例如：重新编译过、文件已加入工程、头文件已包含

7. 如果能提供，最好再附上：
   - 报错截图
   - 相关代码截图
```

## 最少也要提供这 3 样

如果时间很赶，至少发这三个：

1. 第一条 `error` 原文
2. 你改了哪个文件
3. 你想实现题面的哪一部分功能

## 一个好的反馈示例

```text
报错文件：App/app.c
第一条 error：App\\app.c(128): error C208: '_SEG_SetCode': too many actual parameters
我刚改的文件：App/app.c 和 BSP/bsp_seg.h
我改的内容：想把第十四届题目的 E 字母显示补进去
当前情况：主模板能编译，我是把自己的修改加进去后报错
我已检查：重新编译过，函数声明和定义可能还没完全同步
```

## 一个不太好排查的反馈示例

```text
学长我又报错了，帮我看下
```

这种信息太少，别人只能从头猜，会很浪费时间。

## 推荐自己先做的 3 步

在反馈之前，建议先自己试：

1. 只看第一条 `error`
2. 看最近刚改的几行
3. 看 `.h` 和 `.c` 有没有同步

如果还是不行，再把信息按上面的格式发出来。
