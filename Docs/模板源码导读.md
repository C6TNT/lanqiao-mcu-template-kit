# 模板源码导读

这份文档是给第一次读这套模板的人准备的。

目标不是一下子把所有原理讲透，而是先解决两个问题：

1. 这份代码应该从哪里开始看
2. 每一段代码大概是在比赛里负责哪一类功能

## 一、先记住一个总原则

这套模板不是按“外设一个文件、业务乱写一团”的方式组织的，而是分成了几层：

- `main.c`
  只负责程序入口
- `App/app.c`
  只负责业务逻辑
- `BSP`
  只负责开发板资源
- `Devices`
  只负责把常见器件封装成函数
- `Drivers`
  只负责最底层时序

真正刷题时，最常改的是：

- `App/app.c`

最少改的是：

- `Drivers`
- 大部分 `BSP`

## 二、第一次阅读建议顺序

### 第 1 步：看 `main.c`

只需要知道两件事：

- 程序先执行 `App_Init()`
- 然后一直执行 `App_Loop()`

也就是说，整套模板真正的“比赛主逻辑”都在：

- `App_Init()`
- `App_Loop()`

### 第 2 步：看 `App/app.c` 最上面的全局变量

先不要急着看函数，先看变量分成了哪几类：

- 页面变量
- 参数变量
- 实时数据变量
- 报警状态变量
- 平滑后的显示变量

如果把这些变量先看懂，后面页面显示和按键处理就会顺很多。

### 第 3 步：看 `App_ShowXXXPage`

这一组函数负责回答：

- 每个页面显示什么
- 每个页面的数据来自哪

常见规律是：

- 数据页显示“当前采样值”
- 参数页显示“当前阈值”
- 时间页显示“RTC 时间”
- 频率页显示“当前频率”
- 记录页显示“状态量 / 辅助量 / 最近一次事件”

### 第 4 步：看 `App_HandleKey`

这一段负责回答：

- 当前按键按下后会发生什么
- 普通模式和参数编辑模式有什么区别

比赛里很多题本质上都是这里在变。

### 第 5 步：看 `App_Loop`

这是最重要的一步。

`App_Loop()` 决定了：

- 什么时候扫按键
- 什么时候采样
- 什么时候更新报警
- 什么时候刷新时间

这也是综合题最核心的地方。

## 三、`App_Loop()` 应该怎么理解

当前模板用了分时任务的思路：

- `10ms`
  处理按键和高级按键事件
- `100ms`
  采样温度、ADC、距离
- `500ms`
  更新频率和报警闪烁
- `1s`
  更新时间

你可以把它理解成：

- 快任务：按键、闪烁
- 中任务：传感器采样
- 慢任务：时间、统计、记录

比赛里只要会这样拆，很多综合题就不会乱。

## 四、每个区域在赛题里通常对应什么

### 1. 全局变量区

通常对应：

- 页面编号
- 参数项编号
- 实时数据
- 输出状态
- 标志位

### 2. 参数存取区

通常对应：

- 参数保存
- 掉电记忆
- 上电恢复默认值

### 3. 页面显示区

通常对应：

- 数码管显示要求
- 页面编号显示
- 数据格式显示
- 子页面切换

### 4. 参数编辑区

通常对应：

- 参数加减
- 参数切换
- 参数范围限制

### 5. 按键处理区

通常对应：

- S4 切页面
- S5 切子页面
- S8 加
- S9 减
- 长按 / 双击 / 清零

### 6. 状态判断区

通常对应：

- 是否报警
- 是否接近
- 是否超限
- 是否达到继电器触发条件

### 7. 主循环任务区

通常对应：

- 采样节拍
- 刷新节拍
- 输出控制节拍
- 状态更新节拍

## 五、初学者最容易卡住的点

### 1. 变量太多

解决方法：

- 不要一次全记
- 先只分清“页面变量 / 参数变量 / 数据变量”

### 2. 看不出哪个函数对应题面哪一条

解决方法：

- 配合 [赛题要求与代码对应表](./赛题要求与代码对应表.md) 一起看

### 3. 不知道应该先改哪里

经验上最稳的顺序是：

1. 改页面
2. 改按键
3. 改参数
4. 改采样
5. 改输出

## 六、最推荐的实践方式

不要只读代码。

最有效的方式是：

1. 先读这份导读
2. 再看 `App/app.c`
3. 然后拿一道题面
4. 自己标出“这题哪些要求要改页面、哪些要改按键、哪些要改参数”
5. 最后再动手改

这样读代码会比从第一行硬啃快很多。
