# Examples 说明

这里放的是“按当前稳定通用模板重写过的省赛业务层示例”。

这些示例统一遵守当前模板已经验证通过的资源分配：

- `Timer1`：系统时基
- `Timer0`：频率计数
- `Timer2`：串口
- `PCA`：超声波

## 先看这个提醒

这些示例代码的主要作用是：

- 帮助理解“某一届真题怎样落到当前模板里”
- 帮助学习“页面、按键、参数、输出、调度”应该怎么组织
- 帮助练习“如何从基础模板长出完整业务层”

不要把 `Examples` 里的 `.c` 文件直接当成已经全部上板验证过的最终标准答案。

当前更稳妥的说法是：

- 示例代码整体方向和结构是对的
- 部分页次已经根据官方题面截图做了较严格复原
- 但 `Examples` 这一整套并不是每一届都做过完整上板逐项验证

尤其是：

- 第九届
- 第十届
- 第十一届

这几届更应该理解为“模板版示例”和“学习版复盘代码”，不是“可以不检查直接交比赛”的版本。

更准确地说，其实现在 `Examples` 里的所有示例，都建议在使用过程中边学习、边替换、边编译、边上板验证。

如果在学习或使用过程中发现：

- 页面和题面不一致
- 按键行为不一致
- 显示细节不一致
- 输出灯或继电器逻辑不一致
- 编译或上板现象不对

就继续反馈给我，我再按题面和现象往下修。

## 怎么看

每一届现在都配了两份文件：

- `.c`
  看最后业务层代码怎么写
- `.md`
  看这份代码是怎么从基础模板一步步加出来的

最推荐的阅读顺序是：

1. 先看对应年份的 `.md`
2. 先理解这题怎么拆页面、参数、按键、输出
3. 再看对应 `.c`
4. 最后回到主工程 [app.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/App/app.c) 自己重写一遍

## 最后提醒

用这些示例学习时，推荐固定做这几件事：

1. 先读题面。
2. 再读对应 `.md`。
3. 再看对应 `.c`。
4. 自己改进主工程。
5. 编译验证。
6. 上板验证。
7. 发现问题就继续修。

这样学得最扎实，也最接近比赛真正需要的能力。
