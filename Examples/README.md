# Examples 说明

建议先看：

- [模板源码导读](../Docs/模板源码导读.md)
- [赛题要求与代码对应表](../Docs/赛题要求与代码对应表.md)

这里放的是“按当前稳定模板重写过的省赛业务层示例”。

这些示例主要用于：

- 学习某一届题目怎么落到当前模板里
- 练习“页面、按键、参数、采样、输出”怎么组织
- 帮助从基础模板一步步长出完整业务层

## 统一阅读方式

现在每一届的 `.md` 都统一成了同一种结构：

- 这题主要练什么
- 先改哪 5 处
- 这题最容易卡住的地方
- 建议验证顺序

推荐阅读顺序：

1. 先看对应年份的 `.md`
2. 再看对应年份的 `.c`
3. 最后回到 `App/app.c` 自己重写一遍

## 使用提醒

不要把 `Examples` 里的代码直接理解成“已经全部完整验收的最终标准答案”。

更准确的定位是：

- 学习版
- 复盘版
- 模板迁移版

尤其是：

- 第九届
- 第十届
- 第十一届

更建议边学习边编译、边上板验证。

实际上，现在所有示例代码都建议在使用过程中同步核对题面、同步验证现象。如果发现不一致，可以继续反馈并迭代修改。
