# 第十五届示例说明

对应代码：[app_provincial_15.c](D:/桌面/2023蓝桥杯省国赛训练过程/蓝桥杯单片机通用模板工程/Examples/app_provincial_15.c)

## 这次为什么重写
你这次给的是第十五届省赛官方题面截图，所以这版不再沿用之前那种“根据公开复盘整理出来的大致版”，而是按题面里的页面、参数、回显和 LED 规则重写成更贴原题的模板版。

这次最关键的修正有：

- 页面改成题面里的 `频率 / 参数 / 时间 / 回显`
- 参数页拆成 `P1` 和 `P2`
- 回显页拆成 `HF` 和 `HA`
- 频率校准结果为负时，频率页显示 `LL`
- DAC 输出按题面图的分段关系计算
- `L1 / L2` 的含义改成题面定义，而不是旧示例里的泛化逻辑

## 题目主干

第十五届的主线其实很明确：

- `P3.4/T0` 测量 `NE555` 输出频率
- 频率结果需要叠加校准值 `P2`
- 频率超限参数是 `P1`
- 数码管要完成 4 个主页面和 2 组子页面显示
- `DS1302` 提供当前时间
- 需要记录最大频率及发生时间
- `PCF8591` 负责 `DAC` 输出
- `LED` 只重点考 `L1` 和 `L2`

这题的难点不在“测一次频率”，而在“围绕频率，把参数、DAC、时间、回显和报警串成完整业务”。

## 我是怎么从当前基础模板一步步改出来的

### 第 1 步：先按题面拆主页面

先把页面直接按题面命名：

- `PAGE_FREQ`
- `PAGE_PARAM`
- `PAGE_TIME`
- `PAGE_ECHO`

这样后面写显示和按键时，就不用把题面再翻译成别的页面名。

### 第 2 步：再把参数页和回显页拆成子页面

第十五届不是普通“四个页面”这么简单，它有两组子页面：

- 参数页：`P1 / P2`
- 回显页：`HF / HA`

所以我又单独拆了：

- `PARAM_LIMIT`
- `PARAM_CAL`
- `ECHO_FREQ`
- `ECHO_TIME`

这样 `S5` 的行为就会很清楚：

- 在参数页下，切 `P1 / P2`
- 在回显页下，切 `HF / HA`

### 第 3 步：先统一“最终频率”的定义

题面写得很清楚：

- 最终频率 = 实测频率 + 校准值

所以我先单独做了：

- `App_GetFreqValue()`
- `App_UpdateFreqValue()`

后面的显示、DAC、报警、最大值统计，全部围绕这个“最终频率”展开。

### 第 4 步：再单独写 DAC 规则

题面图给了 `DAC` 和频率的对应关系，所以我把它独立成：

- `App_UpdateDac()`

逻辑是：

- 校准后频率小于 0：`DAC = 0V`
- `0 ~ 500Hz`：`DAC = 1V`
- `500Hz ~ Pf`：线性上升到 `5V`
- `>= Pf`：`DAC = 5V`

这样做的好处是，后面如果只想检查 `DAC` 算得对不对，只要盯这一个函数。

### 第 5 步：再把最大值统计单独拿出来

题面要求回显最大频率和发生时间，所以我单独做了：

- `App_UpdateMaxRecord()`

它只在两个条件下更新：

- 当前最终频率不是负数
- 当前最终频率大于历史最大值

更新时同时保存：

- `freq_max`
- `max_time`

### 第 6 步：再按题面写 LED

题面里的 LED 很精确：

- `L1`：频率界面下，0.2 秒闪烁
- `L2`：当前频率大于超限参数时，0.2 秒闪烁
- 若校准后为负，`L2` 常亮

所以我没有把 LED 混在页面显示里，而是集中写到：

- `App_UpdateLed()`

这样页面变了、频率变了、参数变了，都只需要重新调用这一处。

### 第 7 步：最后再写页面显示

等业务规则都定下来后，再去写显示函数：

- `App_ShowFreqPage()`
- `App_ShowParamPage()`
- `App_ShowTimePage()`
- `App_ShowEchoPage()`

其中最关键的细节是：

- 频率页编号就是 `F`
- 参数页编号是 `P1 / P2`
- 回显页编号是 `HF / HA`
- 校准后为负时，频率页最后两位显示 `LL`

### 第 8 步：最后挂回当前模板调度

这版没有回到旧式阻塞测频写法，而是继续沿用当前稳定模板：

- `10ms`：`FREQ_PollTask()` + 按键扫描
- `100ms`：取频率结果、算最终值、更新最大值、更新 DAC、更新 LED 节拍
- `1s`：读取 RTC
- 主循环每圈：刷新数码管显示

这样页面切换和数码管不会被测频逻辑卡住。

## 这份示例里保留了哪些题面重点

- 频率页 `F`
- 参数页 `P1 / P2`
- 时间页 `hh-mm-ss`
- 回显页 `HF / HA`
- `S4` 主页面切换
- `S5` 在参数页和回显页切子页面
- `S8 / S9` 调整参数
- `Pf` 范围 `1000 ~ 9000`
- 校准值范围 `-900 ~ 900`
- 校准后频率为负时显示 `LL`
- 最大频率值和对应时间回显
- `DAC` 按题面图分段输出
- `L1 / L2` 按题面要求工作

## 这版和之前第十五届示例相比，最重要的变化

- 参数页改成了明确的 `P1 / P2`
- 回显页改成了明确的 `HF / HA`
- 时间页显示改成完整的 `时-分-秒`
- 频率错误态改成 `LL`
- `DAC` 规则改成题面图那套
- `LED` 逻辑改成官方题面版

## 最值得先看的几块

推荐顺序：

1. `App_UpdateFreqValue()`
2. `App_UpdateDac()`
3. `App_UpdateMaxRecord()`
4. `App_UpdateLed()`
5. `App_HandleKey()`
6. `App_Loop()`

这几块正好把第十五届最核心的东西串起来了：

- 最终频率怎么定义
- DAC 怎么联动
- 最大值怎么记
- LED 怎么响应
- 页面和子页面怎么切
- 怎么挂回稳定模板
